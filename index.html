Movie Night Planner

Individual Project

Movie Night Planner is a web application that allows users to plan and organize movie nights with their friends or family. Users can search for movies, get suggestions, and save their movie night details for future reference. The application uses an external movie database API to fetch movie data based on user input and provides a user-friendly interface for planning movie nights.

Front-End Technology Stack:

  • HTML5 / CSS3
  • JavaScript
  • Parcel: A build tool that supports fast and efficient asset bundling and development environment setup.
  • External Movie Database API: The application fetches movie data from an external API to provide movie suggestions based on user input.

Key Features

  • Movie Search: The application allows users to search for movies by entering the movie title. In the research box, it fetches the movies using the user's input and display every movie that could correspond to the research term. In the planning and watch party pages, it fetches movie suggestions from the external database and displays them as a dropdown for selection.
  • Movie Night Planning: Users can plan their movie nights by selecting a movie from the suggestions, entering the date and time of the movie night, and adding any notes or comments related to the event.
  • Local Storage: The application uses local storage to save the planned movie nights, allowing users to view and access their movie night details even after refreshing the page. We also use local storage to save the favorite and watch lists of the user. Using this data we can then create personalized movie suggestions.
  • Movie Lists: Creation of various movie lists for the home page, such as movies currently in theaters, top-rated movies, family movies, action movies, etc. by doing GET requests.
  • User-Friendly Interface: The application provides an intuitive and easy-to-use interface, making it convenient for users to plan and manage their movie nights with ease.

Self-evaluation

Upon evaluating our website, I believe it deserves a grade of A. We were able to apply the knowledge we gain during the semester, working with objects, local-storage and apis requests. The core functionalities, such as randomly selecting movies for watch parties and playing trailers, work smoothly, and were fun to create. Overcoming the challenges of making everything work together seamlessly was a significant achievement for our team. I am personally really glad to have contributed ideas and innovative solutions to the project, leveraging the knowledge I gained during the learning process. It was a rewarding experience to witness the website come to life and be able to create an enjoyable and interactive platform.

Sleep-Outside

Group Project

Sleeping Outside is a E-Commerce website that aims to offer a wide range of outdoor sleeping products, including tents, sleeping bags, backpacks, and hammocks. The website provides a user-friendly shopping experience, thanks to its single-page application (SPA) architecture, which allows users to explore different products without the need for page reloads. The product data is retrieved from a server.

Front-End Technology Stack:

  • Node.js
  • JavaScript
  • Vite: A build tool that supports fast and efficient asset bundling and development environment setup.
  • HTML5 / CSS3
  • Jwt-decode: A JavaScript library used for decoding JSON Web Tokens on the client-side.

Key Features

  • Product Pages: The website has dedicated pages for tents, sleeping bags, backpacks, and hammocks. Each product category page provides a list of relevant products along with their images, descriptions, and prices.
  • Product Detail Page: When a user clicks on a product from any category page, they are taken to the product detail page. Here, they can view more detailed information about the selected product, such as specifications, customer reviews, and related items.
  • Shopping Cart: Users can add products to their shopping cart, which displays the selected items and their total prices. Users can easily manage their cart contents and proceed to checkout.
  • Authentication with JWT-decode: User authentication is implemented using JSON Web Tokens (JWTs). Upon successful authentication on the server-side, the server generates a JWT, which is sent to the client-side. The website uses the jwt-decode library to extract user information from the JWT and manage authenticated user sessions.
  • Server Data Retrieval: The product information is fetched from the server using both GET and POST requests. GET requests are used to fetch product details for display on various pages, while POST requests are specifically employed during the checkout process. These POST requests are responsible for transmitting user's order information securely to the server for processing, enabling the website to handle large inventories efficiently and provide the most up-to-date product data.
